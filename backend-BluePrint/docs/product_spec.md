## 1. Product Overview

**WMS Platform** — универсальная облачная SaaS-система управления складом нового поколения. Продукт закрывает полный операционный цикл склада: приёмка, хранение, сборка заказов, отгрузка, инвентаризация — с мощной аналитикой и глубокими интеграциями.

### Target Segments

| Сегмент | Проблема | Наше решение |
|---------|----------|--------------|
| Фулфилмент-центры (3PL) | Excel + 1С не справляется с ростом | Мультиклиентский WMS с биллингом хранения |
| Интернет-магазины | Нет единого учёта склад + заказы + маркетплейсы | Единая платформа с WB/Ozon/ЯМ коннекторами |
| Ритейл (склад РЦ) | Legacy WMS стоит $500k внедрения | SaaS за $299/мес, запуск за 1 день |
| Логистические компании | Нет прозрачности операций | Real-time дашборды и KPI сотрудников |
| Производство | Учёт сырья/готовой продукции разрознен | Партионный учёт + производственные заказы |

## 2. Feature Map (MoSCoW)

### Must Have (MVP)

- Управление структурой склада (зоны, ряды, ячейки)
- QR/штрихкод-генерация ячеек и товаров
- CRUD товаров: SKU, штрихкод, характеристики
- Остатки в реальном времени по ячейкам
- Операции: Приёмка, Размещение, Перемещение, Отгрузка
- Управление заказами (inbound/outbound)
- Система задач для сотрудников
- Android Mobile App (сканирование + задачи)
- Базовая аналитика и отчёты
- Multi-tenancy, RBAC, аудит-лог
- REST API

### Should Have (Q2)

- Инвентаризация (full + cycle count)
- Пакетная сборка (batch/wave picking)
- Интеграция 1С
- Wildberries + Ozon коннекторы
- KPI сотрудников
- Прогноз остатков
- Партионный учёт + сроки годности

### Could Have (Q3-Q4)

- AI-прогнозирование спроса
- Slotting optimization (умное размещение)
- iOS приложение
- ERP коннекторы (SAP, Odoo)
- White-label

### Won't Have (Now)

- Производственные заказы (WO)
- TMS (Transportation Management)
- Бухгалтерский учёт (не замена 1С)

## 3. Functional Requirements

### FR-001: Warehouse Structure

- Система поддерживает иерархию: Склад → Зона → Ряд → Стеллаж → Полка → Ячейка
- Ячейка имеет уникальный код, QR, штрихкод, габариты, ограничения
- Bulk-создание ячеек по шаблону (A-01 → A-50)
- Печать маркировки ячеек в PDF

### FR-002: Inventory Management

- Остатки хранятся с точностью до ячейки + партии + серийного номера
- Поддержка FEFO, FIFO, LIFO стратегий списания
- Резервирование товара под заказ
- Минимальные остатки с автоуведомлением

### FR-003: Operations

- Каждая операция создаёт запись inventory_movement
- Все операции проходят через задачи (tasks)
- Задачи назначаются автоматически или вручную
- Сотрудник может выполнить операцию только после сканирования

### FR-004: Mobile App

- Работает offline (последние 100 задач кэшируются)
- Синхронизация автоматическая при появлении сети
- Поддержка камеры смартфона и аппаратных сканеров ТСД
- PIN-код аутентификация

### FR-005: Analytics

- Все события пишутся в аналитическую БД асинхронно
- Дашборды обновляются в реальном времени (WebSocket)
- Экспорт отчётов в Excel, PDF, CSV

## 4. Non-Functional Requirements

| Требование | Метрика |
|------------|---------|
| API Response Time (p95) | < 200ms |
| API Response Time (p99) | < 500ms |
| Uptime SLA | 99.9% (8.7 ч/год downtime) |
| Mobile Scan Latency | < 100ms от скана до ответа |
| Concurrent Users per Tenant | до 500 |
| Data Isolation | Полная изоляция между тенантами |
| GDPR / 152-ФЗ | Данные в EU или RU по выбору |
| Backup | Daily snapshot + point-in-time recovery |
| RTO | 4 часа |
| RPO | 1 час |
